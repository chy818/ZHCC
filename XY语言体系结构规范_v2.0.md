## XY 语言体系结构规范 (v0.1)

0. 前言与设计目标

0.1 设计哲学
本版本 (v0.1) 的核心目标是“自举优先”与“中文原生”。

- 自举优先：一切特性以“能用 XY 语言写出 XY 编译器”为最高优先级。为此，牺牲部分高级特性（如完整泛型、模式匹配），换取实现的确定性和简洁性。
- 中文原生：标识符、关键字、错误信息全面支持中文，消除非英语母语者的认知摩擦。
- 简化内存：v0.1 不引入复杂的垃圾回收 (GC)，采用栈分配 + 确定性析构模型，确保资源管理的可预测性。
- LLVM 亲和：底层直接映射到 LLVM IR，利用其优化能力和跨平台特性。

0.2 术语表
术语 定义/说明
文本 对应 UTF-8 字符串 (String)。规范中统一使用“文本”，避免混用“字符串”。
空 无返回值类型，等价于 ()。
指针 不透明内存地址类型 (void\*)，仅用于 FFI 和内部实现，用户代码不可解引用。
列表 动态数组容器。v0.1 通过类型擦除实现，底层存储 指针，用户需自行转换。
运行时库 由 C 语言编写的基础库 (rt.c)，提供内存分配、文本操作等底层功能。
确定性析构 编译器在变量离开作用域时，自动插入对应的资源释放代码。
自举子集 语言的一个子集，足以编写编译器本身。

1. 语言核心规范 (Core Language)

1.1 词法与语法

- 标识符: 允许任意非空白、非运算符的 Unicode 字符序列。
- 关键字 (Keywords):
  - 严禁用作标识符。
  - 结构控制: 函数, 结构体, 枚举, 外部, 导入, 若, 否则, 循环, 当, 返回, 中断, 继续.
  - 类型相关: 整数, 浮点, 布尔, 文本, 空, 列表, 指针.
  - 预留: 让 (v0.1 暂不支持，报错提示“预留特性”), 可变, 自.
- 字面量:
  - 布尔值：真, 假 (不作为关键字，作为保留字面量)。
  - 整数：123, -45.
  - 文本："内容".
- 注释: 支持 // 和 /\* \*/，完全支持 UTF-8。

1.2 类型系统

- 基础类型: 整数 (i64), 浮点 (f64), 布尔, 文本, 空, 指针.
- 复合类型:
  - 结构体: 结构体 名称 { 字段：类型，... }.
  - 枚举: 枚举 名称 { 变体 1 (类型 1, 类型 2...), 变体 2, ... }.
    - 重要变更: 变体支持携带任意数量的字段（元组风格）。
    - 示例：枚举 表达式 { 数字 (整数), 加法 (左：节点，右：节点，操作符：文本) }.
  - 列表: 列表 (动态数组).
    - 实现机制: v0.1 采用类型擦除。底层存储 指针。
    - 使用方式: 用户存入数据前需隐式或显式转换为 指针 (对于结构体/文本)，取出时转回具体类型。
    - 注: 为简化自举，初期可提供 列表*整数, 列表*节点 等特化版本，或统一使用 列表 + 强制转换。规范推荐统一 列表 API，内部存 指针。
- 传递语义:
  - 默认按值传递。
  - 后端优化: 编译器后端可根据 LLVM 的 byval 属性自动优化大结构体的传递方式，但语言层面不暴露引用符号 (&)。

1.3 内存模型

- 栈分配: 局部变量默认 alloca。
- 堆管理与确定性析构:
  - 文本 和 列表 的内部数据在堆上分配。
  - 编译器责任: 编译器必须进行作用域分析。当变量离开作用域（函数返回、块结束）时，必须生成调用运行时释放函数的代码。
  - 示例：let s = "hello"; 在块结束时自动生成 rt_string_free(s)。
  - 限制: 开发者应避免创建循环引用数据结构（如父子节点互指），因无 GC，这将导致内存泄漏。

1.4 内置函数与运行时 API
编译器隐式链接运行时库，并提供以下高层封装（用户可直接调用）：

- 打印 (内容：文本) -> 空
- 新建列表 () -> 列表
- 列表追加 (list: 列表，item: 指针) -> 空 (注意 item 为指针)
- 列表获取 (list: 列表，index: 整数) -> 指针
- 读取行 () -> 文本
- 文本长度 (t: 文本) -> 整数
- 注: 具体的类型转换（如 整数 转 指针）可通过 FFI 或内置转换函数处理，或在 IR 层直接 bitcast。

2. 自举子集规范 (Bootstrap Subset)

2.1 必须支持的特性清单 (S\_{min})
V1 (Rust) 必须实现，V2 (XY) 仅限使用：

1.  控制流: 如果/否则, 循环, 返回, 中断, 继续.
2.  数据定义:
    - 结构体 (多字段).
    - 枚举 (多字段变体，必需).
    - 常量.
3.  模块化:
    - 导入 "相对路径": 将目标文件内容文本包含到当前作用域（类似 C #include），无命名空间隔离。
    - 路径解析：相对于当前文件目录。
4.  FFI:
    - 支持声明并调用任意 C 函数（包括 malloc, printf, fopen 等）。
    - 支持 指针 类型。
5.  递归: 支持函数递归调用。
6.  类型: 整数, 布尔, 文本, 指针, 空, 列表 (指针版).

2.2 自举合规性测试

- 构建流程:
  1.  xyc_rust --input src/compiler_v2.xy --output xyc_v2 (使用 V1 编译 V2 源码).
  2.  ./xyc_v2 --input tests/hello.xy --output hello (使用 V2 编译测试程序).
  3.  运行 ./hello 验证输出。
- 自动化检查:
  - CI 脚本需扫描 V2 源码，正则匹配禁止的关键字（如 泛型, 特质 等未实现特性），发现即报错。

3. 代码生成规范 (Code Generation)

3.1 标识符映射规则

- 适用范围: 仅用户定义的标识符。编译器生成的临时变量（%tmp0）保持 ASCII。
- 逻辑:
  1.  若匹配 ^[a-zA-Z\_][a-zA-Z0-9_]\* rightarrow 直接输出 (@foo).
  2.  否则 rightarrow 包裹双引号，转义 为 ，" 为 "。
      - 格式：@"名称" 或 %"名称" (无空格).
- 示例修正:
  - 让 my-var = 10 rightarrow % "my-var" = alloca i64 (错) rightarrow %"my-var" = alloca i64 (对).
  - 函数 计算 () rightarrow define void @"计算"().

3.2 程序入口点

- 优先级: 优先查找 主 函数。若不存在，查找 main (作为普通函数，不自动作为入口，除非用户显式指定链接选项)。若两者皆无，报错。
- 包装逻辑:
  - 生成全局 @main。
  - 若 主 返回 空: call @"主"(); ret i32 0.
  - 若 主 返回 整数: ret call @"主"().
  - 若 主 返回其他: 编译错误。
- 一致性检查: 编译器需检查 主 函数体内所有 返回 语句的类型是否与函数签名一致。

3.3 调试信息 (DWARF)

- 策略: Name 字段始终为原始 UTF-8 中文名。
- IR 示例:
  !0 = !DISubprogram(name: "计算面积", linkageName: @"计算面积", ...)
  define void @"计算面积"() !dbg !0 { ... }
- 稳定模式 (--stable-symbols):
  - linkageName 变为哈希值 (如 "\_cn_a1b2").
  - name 仍为 "计算面积".
  - IR 引用使用哈希名：call void @\_cn_a1b2().

4. FFI 与扩展规范

4.1 外部函数声明

- 语法: 外部 函数 名称 (参数列表) -> 返回类型 [链接名];
- 类型限制:
  - 参数/返回值仅支持：整数, 浮点, 布尔, 指针, 文本 (映射为 i8\*).
  - 结构体: v0.1 不建议直接按值传递复杂结构体给 C，建议传 指针。
- 链接名规则:
  - 若名称含非 ASCII，必须显式指定 链接名。
  - 若省略且名称为 ASCII，默认同名。
- AI 占位符:
  - #[ai("描述")]
  - 行为: 若函数体为空，编译器生成调用 abort() 或抛出 "Unimplemented AI Feature" 的代码，并给出警告。严禁静默忽略，以防运行时错误难以排查。

5. 工具链与构建规范

5.1 项目结构与运行时

- 结构:
  src/main.xy # 入口
  src/lib/\*.xy # 模块
  runtime/rt.c # 运行时源码
  runtime/rt.h # 头文件
- 运行时库接口 (附录 A): rt.c 必须实现以下函数供编译器调用：
  - void* rt_string_new(const char* utf8_str);
  - void rt_string_free(void\* str);
  - void\* rt_list_new();
  - void rt_list_append(void* list, void* item);
  - void* rt_list_get(void* list, int index);
  - void rt_list_free(void\* list);
  - void rt_println(void\* str);

5.2 编译流程

1.  前端: xyc 解析 .xy rightarrow LLVM IR (.ll).
2.  汇编: llc 或 LLVM API rightarrow$ .o.
3.  运行时编译: gcc -c runtime/rt.c -o rt.o.
4.  链接: clang main.o rt.o -lm -o app (注意链接数学库等依赖).

5.3 错误处理

- 格式: [错误] 文件：行：列 - 描述
- 示例: [错误] parser.xy:105:12 - 语法错误：枚举变体 '加法' 期望字段列表，但发现 '}'
- 要求: 全程中文，定位精确。
